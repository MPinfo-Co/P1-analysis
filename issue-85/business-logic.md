# 業務邏輯分析：SA Agent 活文件測試 0415

## 需求說明
測試需求：新增公司設備盤點功能，需記錄設備名稱、序號、負責人，並支援匯出報表。

## 商業邏輯（選填）

### 設備盤點 CRUD 規則

1. **新增設備**：序號（serial_number）為唯一識別鍵，新增時需驗證序號不重複；重複時回傳錯誤提示。
2. **負責人指派**：負責人對應 `users` 資料表中的有效使用者（`is_active = TRUE`）；若指定人員已停用，前端下拉選單不顯示，但既有紀錄保留原值（不強制清空）。
3. **刪除設備**：直接實體刪除（無軟刪除需求），操作不可復原。
4. **匯出報表**：
   - 匯出當前篩選條件下的全部設備清單（非分頁資料）。
   - 匯出格式：Excel（`.xlsx`），欄位順序：設備名稱、序號、負責人姓名、建立時間。
   - 後端產生後以 HTTP 串流回傳，前端觸發下載，不存檔至伺服器。

### 權限控制

| 操作 | 所需權限 |
|------|----------|
| 查看設備清單 | 登入即可 |
| 新增 / 編輯 / 刪除 | 待 role 模組上線後補定義，目前暫不設限 |
| 匯出報表 | 同查看 |

## 資料模型示意（選填）

> 比對 schema.md（v2，2026-03-31）：現有 schema 無設備相關資料表，新增 `company_assets`。

### 新增資料表：company_assets

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | BIGSERIAL, PK | 主鍵 |
| device_name | VARCHAR(200), NOT NULL | 設備名稱 |
| serial_number | VARCHAR(200), NOT NULL, UNIQUE | 設備序號，不可重複 |
| owner_user_id | INTEGER, NULLABLE, FK → users | 負責人；NULL 表示尚未指派 |
| created_at | TIMESTAMP, NOT NULL, DEFAULT NOW() | |
| updated_at | TIMESTAMP, NOT NULL, DEFAULT NOW() | |

### 關聯說明

```
users (1) ──< company_assets (N)
  id            owner_user_id
```

- 一位使用者可負責多筆設備（一對多）。
- 負責人為選填；設備可在無負責人的情況下建立。

## SD 注意事項（選填）

1. **匯出 Excel**：使用 `openpyxl` 產生 `.xlsx`，FastAPI 以 `StreamingResponse`（`media_type="application/vnd.openxmlformats-officedocument.spreadsheetml.sheet"`）回傳，Content-Disposition 設為 `attachment; filename="assets_export.xlsx"`。不引入 `pandas`，避免增加不必要的依賴。
2. **序號唯一性**：資料庫層加 UNIQUE constraint，後端捕捉 `IntegrityError` 並回傳 `400 + { detail: "序號已存在" }`，前端依此顯示欄位錯誤。
3. **負責人下拉**：前端呼叫 `GET /users?is_active=true` 取得有效使用者清單（複用既有 API 或由 SD 確認是否需新增精簡版端點）。
4. **匯出資料量**：若設備筆數預期可能超過 10,000 筆，SD 評估是否需改為非同步產檔（Celery Task + 下載連結），目前需求規模小，暫以同步串流即可。

### 新增 / 編輯 Modal

```
┌─────────────────────────────┐
│ 新增設備                  ✕  │
├─────────────────────────────┤
│ 設備名稱 *                   │
│ [ 請輸入設備名稱           ] │
│                              │
│ 序號 *                       │
│ [ 請輸入設備序號           ] │
│ ⚠ 序號已存在（錯誤提示）      │
│                              │
│ 負責人                       │
│ [ 下拉選單：有效使用者     ▼] │
│                              │
│           [取消]  [儲存]     │
└─────────────────────────────┘
```

### 操作流程

```
使用者點擊「新增設備」
  → 開啟 Modal
  → 填寫欄位並送出
  → 後端驗證序號唯一性
      ├─ 失敗 → Modal 顯示欄位錯誤，不關閉
      └─ 成功 → Modal 關閉，清單刷新

使用者點擊「匯出」
  → 後端依當前篩選條件查詢全部設備
  → 產生 .xlsx 串流回傳
  → 瀏覽器自動下載 assets_export.xlsx
```
