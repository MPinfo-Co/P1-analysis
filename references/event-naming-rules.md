# 安全事件命名規則

> 適用：Gemini Pro Stage 2 system prompt | 維護者：MIS

## 命名格式

```
[事件類型] — [簡述]
```

- 事件類型從下表選擇；簡述不超過 30 字，需包含關鍵設備/IP/行為
- 標題不帶風險等級（已有獨立的 star_rank 欄位）
- 破折號用全形「 — 」（前後各一空白）

## 事件類型清單

| 類型名稱 | 適用場景 |
|---------|---------|
| DNS C2 通訊 | DNS 隧道、Beaconing、C2 心跳連線 |
| 暴力破解 | 密碼嘗試、帳號鎖定、分散式低速破解 |
| 防火牆策略異常 | 規則過寬、非預期放行、策略衝突 |
| VPN 異常流量 | VPN 端點異常封包、DNS 注入、隧道濫用 |
| 惡意程式感染 | 已知惡意程式特徵、可疑執行檔、後門 |
| 資料外洩 | 大量資料傳輸至外部、異常上傳行為 |
| 帳號異常 | 非預期登入、權限提升、休眠帳號活動 |
| 橫向移動 | 內網掃描、SMB 擴散、Pass-the-Hash |
| 環境異常 | GeoIP 封鎖紀錄、大量掃描、正常防禦運作 |
| 設定變更 | 防火牆規則異動、AD 群組變更、排程修改 |

## Prompt 嵌入片段

```
你在產出安全事件時，每個事件的標題必須遵循以下格式：
[事件類型] — [簡述]

事件類型必須從以下清單中選擇：
DNS C2 通訊、暴力破解、防火牆策略異常、VPN 異常流量、惡意程式感染、
資料外洩、帳號異常、橫向移動、環境異常、設定變更

簡述不超過 30 字，需包含關鍵設備 IP 或行為描述。
標題中不要帶風險等級（如嚴重、高危），因為已有獨立的優先級欄位。
```
