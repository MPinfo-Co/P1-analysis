# 處理優先級評定規則

> 適用：Gemini Pro Stage 2 system prompt | 維護者：MIS

## 星等定義

| 星等 | 等級 | 定義 | 處置時效 |
|------|------|------|---------|
| ★★★★★ | 最高危 | 已確認的主動攻擊，系統已遭入侵或正在受攻擊 | 立即 |
| ★★★★☆ | 高危 | 高風險但尚未確認入侵，攻擊面已暴露 | 當日 |
| ★★★☆☆ | 中危 | 異常行為已偵測，需進一步調查 | 本週 |
| ★★☆☆☆ | 低危 | 單一或低頻異常，可能誤報但需觀察 | 列入追蹤 |
| ★☆☆☆☆ | 資訊 | 正常防禦運作紀錄、環境雜訊 | 僅記錄 |

## 四維度評分

| 維度 | 選項 | 分數 |
|------|------|------|
| 攻擊確認度 | 已確認 / 高度疑似 / 待確認 / 可能誤報 | +2 / +1 / 0 / -1 |
| 影響範圍 | 核心基礎設施 / 多台設備 / 單一端點 / 無直接影響 | +2 / +1 / 0 / -1 |
| 發生頻率 | 持續進行中 / 間歇性 / 單次 | +1 / 0 / -1 |
| 關聯性 | 與其他高危事件關聯 / 獨立事件 | +1 / 0 |

**總分對應**：5+→★★★★★ ｜ 3~4→★★★★☆ ｜ 1~2→★★★☆☆ ｜ 0~-1→★★☆☆☆ ｜ -2以下→★☆☆☆☆

## Prompt 嵌入片段

```
你在產出安全事件時，每個事件必須標示處理優先級（1-5 星）。

評定標準基於四個維度：
1. 攻擊確認度：已確認(+2)、高度疑似(+1)、待確認(0)、可能誤報(-1)
2. 影響範圍：核心設施(+2)、多台設備(+1)、單一端點(0)、無直接影響(-1)
3. 發生頻率：持續進行(+1)、間歇性(0)、單次(-1)
4. 關聯性：與其他高危事件關聯(+1)、獨立事件(0)

總分對應：5+→5星、3~4→4星、1~2→3星、0~-1→2星、-2以下→1星

在事件描述的【風險分析】段落中，簡要說明各維度的判定依據。
```
