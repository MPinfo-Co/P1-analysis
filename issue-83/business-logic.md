# 業務邏輯分析：Agent 流程測試 0415

## 需求說明
測試 SA Agent 自動化流程。\n\n需求：建立一個簡單的使用者登入功能，包含帳號密碼驗證。

## 商業邏輯（選填）

### 登入流程

```
使用者輸入帳號與密碼
        ↓
  帳號是否存在？
   ├── 否 → 回傳「帳號或密碼錯誤」（不區分錯誤原因，避免帳號枚舉）
   └── 是 ↓
  密碼是否正確？
   ├── 否 → 累計登入失敗次數
   │         ├── 失敗次數 < 上限 → 回傳「帳號或密碼錯誤」
   │         └── 失敗次數 ≥ 上限 → 鎖定帳號，回傳「帳號已鎖定」
   └── 是 → 建立 Session / 發行 Token，導向首頁
```

### 業務規則

| 規則 | 說明 |
|------|------|
| 錯誤訊息一致性 | 帳號不存在與密碼錯誤統一回傳相同訊息，防止帳號枚舉攻擊 |
| 連續失敗鎖定 | 連續失敗達閾值（建議 5 次）後鎖定帳號，需由管理員或等待冷卻時間解鎖 |
| 密碼不明文儲存 | 資料庫僅存雜湊值，驗證時以相同演算法比對 |

## 資料模型示意（選填）

```
┌─────────────────────────┐        ┌──────────────────────────┐
│          User           │        │         Session          │
├─────────────────────────┤        ├──────────────────────────┤
│ id            (PK)      │ 1    * │ id              (PK)     │
│ username      (UNIQUE)  │────────│ user_id         (FK)     │
│ password_hash           │        │ token           (UNIQUE) │
│ is_locked               │        │ created_at               │
│ failed_attempts         │        │ expires_at               │
│ locked_at               │        └──────────────────────────┘
│ created_at              │
│ updated_at              │
└─────────────────────────┘
```

- `User.is_locked`：帳號是否被鎖定
- `User.failed_attempts`：累計連續失敗次數，登入成功後重設為 0
- `Session.token`：登入成功後產生，用於後續請求的身份識別

## SD 注意事項（選填）

1. **密碼雜湊**：使用 bcrypt（cost factor ≥ 12）或 Argon2id，禁止 MD5 / SHA1 等快速雜湊演算法。
2. **防暴力攻擊**：除帳號鎖定外，建議於 API 層加上 Rate Limiting（例如同一 IP 每分鐘最多 10 次登入請求）。
3. **Session 管理**：若採 JWT，需設定合理的 `exp`（建議 Access Token 15 分鐘、Refresh Token 7 天）；若採 Server-side Session，需考慮儲存媒介（Redis 等）。
4. **HTTPS**：登入端點必須強制 HTTPS，防止憑證明文傳輸。
5. **冷卻解鎖機制**：鎖定後可設定自動冷卻時間（例如 30 分鐘後自動解鎖），避免合法使用者無法自助恢復。

## 畫面示意（選填）

```
┌──────────────────────────────────┐
│            使用者登入             │
├──────────────────────────────────┤
│  帳號                            │
│  ┌────────────────────────────┐  │
│  │ 請輸入帳號                 │  │
│  └────────────────────────────┘  │
│                                  │
│  密碼                            │
│  ┌────────────────────────────┐  │
│  │ ••••••••                   │  │
│  └────────────────────────────┘  │
│                                  │
│  ┌────────────────────────────┐  │
│  │          登 入             │  │  ← 按下後顯示 Loading
│  └────────────────────────────┘  │
│                                  │
│  [錯誤狀態] 帳號或密碼錯誤       │  ← 紅色提示文字
│  [鎖定狀態] 帳號已鎖定，          │
│            請聯繫管理員           │
└──────────────────────────────────┘
```

**操作流程**

1. 使用者進入登入頁，輸入帳號與密碼。
2. 按下「登入」按鈕，按鈕進入 Loading 狀態，防止重複送出。
3. 驗證成功 → 導向應用程式首頁。
4. 驗證失敗 → 於表單下方顯示錯誤訊息，密碼欄位清空，焦點回到密碼欄。
5. 帳號被鎖定 → 顯示鎖定提示，隱藏密碼輸入欄，引導使用者聯繫管理員。
